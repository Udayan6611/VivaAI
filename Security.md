# 🔐 Security Policy

## 📌 Supported Versions

| Version | Supported       |
| ------- | --------------- |
| 1.x.x   | ✅ Supported     |
| <1.0    | ❌ Not supported |

---

## 🎙️ About VivaAI

VivaAI is a next-generation AI-powered interview platform designed to streamline the hiring process through real-time communication and intelligent assessment.

It provides a seamless interviewing experience for recruiters and candidates using:

* Real-time video/audio powered by WebRTC
* AI-driven assessment integrated with SarvamAI

---

## 🎯 Scope

### ✅ In-Scope

* Issues affecting real-time communication (WebRTC)
* Unauthorized access to interview sessions (audio/video)
* Exposure of candidate or recruiter data
* Vulnerabilities in AI-driven assessment workflows
* Authentication or access control issues

---

### ❌ Out-of-Scope

* UI/UX issues without security impact
* Feature requests or performance improvements

---

## 📬 Reporting a Vulnerability

If you discover a security issue, please report it responsibly.

### Please include:

* Description of the issue
* Steps to reproduce
* Impact of the vulnerability
* Supporting evidence (screenshots, PoC if available)

---

## ⏱️ Response Timeline

* Acknowledgement: Within 24 hours
* Further communication will follow after initial review

---

## 🔒 Responsible Disclosure

* Please do not publicly disclose vulnerabilities before they are reviewed and resolved
* Only test on accounts and data you own or have permission to use

---

## ⚖️ Safe Harbor

We support responsible security research conducted in good faith and will work with researchers to address valid issues.

---

## 🙌 Acknowledgements

We appreciate contributions from the security community that help improve VivaAI.
